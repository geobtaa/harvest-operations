ArcGIS Harvester

Read more documentation in arcgis.md.

Purpose

This script scans the APIs of ArcGIS Hubs and returns metadata for all web services. It produces two CSV files:

  • A Primary CSV containing all metadata except the link fields
  • A Distribution CSV with only the link fields (e.g., landing page, download, REST services)

Inputs

Run the Harvest Task Dashboard first. The ArcGIS harvester uses the workflow inputs generated by the harvest task dashboard together with website metadata defaults stored in reference_data/websites.csv.

Outputs

The harvester writes dated output files under outputs/:

  • outputs/YYYY-MM-DD_arcgis_primary.csv: harvested dataset records plus updated harvest-record rows.
  • outputs/YYYY-MM-DD_arcgis_distributions.csv: distribution links derived from the primary output.
  • outputs/YYYY-MM-DD_arcgis_report.csv: per-harvest-record run status, record counts, and totals.

When upload deltas are built, upload-ready CSVs are written under outputs/to_upload/:

  • outputs/to_upload/YYYY-MM-DD_arcgis_primary_upload.csv
  • outputs/to_upload/YYYY-MM-DD_arcgis_distributions_new.csv
  • outputs/to_upload/YYYY-MM-DD_arcgis_distributions_delete.csv
