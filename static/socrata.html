Socrata Harvester

Purpose

This script scans the DCAT APIs of Socrata portals and returns metadata for GIS and map datasets. It produces two CSV files:

  • A Primary CSV containing all metadata except the link fields
  • A Distribution CSV with only the link fields (e.g., landing page, documentation, GeoJSON)

Inputs

Run the Harvest Task Dashboard first. The Socrata harvester uses the workflow inputs generated by the harvest task dashboard together with website metadata defaults stored in reference_data/websites.csv.

  • inputs/harvest-workflow-inputs/py-socrata.csv: harvest-record rows with endpoint and lifecycle fields.
  • reference_data/websites.csv: descriptive website defaults used for harvested dataset records.

Outputs

The harvester writes dated output files under outputs/:

  • outputs/YYYY-MM-DD_socrata_primary.csv: harvested dataset records plus updated harvest-record rows.
  • outputs/YYYY-MM-DD_socrata_distributions.csv: distribution links derived from the primary output, with invalid GeoJSON links removed before writing.
  • reports/socrata/YYYY-MM-DD_socrata_report.csv: per-harvest-record run status, record counts, and totals.
