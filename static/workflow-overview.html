Harvest task workflow

  1. Open Harvest Dashboard and run the Triage task.
  2. Open the Triage view and identify sets to work on next.
  3. Open the set’s Harvest record in GEOMG and add queue:pending_* and due:yyyy-mm-dd tags.
  4. Download the harvest records again and rerun the Triage task.
  5. Open the To do view.
  6. Create a GitHub issue for a task.
  7. Complete the task and close the GitHub issue.
  8. Open the Harvest record, update the Last Harvested field, and remove the tags.

Dashboard tags

Use these tags in the Harvest record’s Tags field. The dashboard recognizes tags separated by a pipe, comma, or semicolon.

  • queue:pending_harvest — queues harvesting work. It places the record in To do and keeps it in the Triage harvest section, including records with irregular periodicity.
  • queue:pending_updates — queues record-update work. It places the record in To do.
  • queue:pending_review — queues a review. It places the record in To do and uses its due date as the review date.
  • due:YYYY-MM-DD — sets the due date for any pending queue tag. harvest_due:YYYY-MM-DD is also accepted. Without either, queued work is due today.
  • review:Ny — schedules an irregular record for review N years after its Last Harvested date; for example, review:2y.
