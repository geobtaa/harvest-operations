Harvest Task Workflow

Back to Harvest Task Dashboard

Use this checklist at the start of each work session and repeat the refresh steps after changing Harvest records.

Regular workflow

  1. Open the Harvest Task Dashboard

    Keep the dashboard open while you work through the remaining steps.

  2. Download both current CSVs

    Use the dashboard links and save the files as inputs/harvest-records.csv and reference_data/websites.csv.

  3. Generate Workflow Inputs

    Select the dashboard button and wait until its status says Completed.

  4. Run Triage

    Select the dashboard button and wait until its status says Completed.

  5. Open Triage and decide what should be worked on next

    Start with Reharvest due. Monthly, quarterly, and annual records need to be reharvested on their schedule; irregular records need review. A highlighted review flag shows when a review:Ny review is upcoming or overdue. Use Last Harvested and Admin Notes as additional clues.

  6. Update the selected Harvest records

    In GEOMG, add the appropriate queue:pending_* tag and a due:YYYY-MM-DD tag. Save and sync each record.

  7. Refresh the dashboard data

    Download both CSVs again, Generate Workflow Inputs again, and Run Triage again. Confirm the selected tasks now appear in To do.

  8. Open To do and create the GitHub issues

    Use the issue button for each new task, then assign the issue to the project lead or to yourself.

Do not create a new issue directly from Triage. Update and sync the Harvest record first, refresh the CSVs and Triage, and then create the issue from To do.

When a task is finished

  1. Update the Harvest record

    Update the Last Harvested field when appropriate, remove the completed queue and due tags, and sync the record.

  2. Close the GitHub issue

    Add any useful completion notes before closing it.

  3. Refresh again

    Repeat the CSV download, Generate Workflow Inputs, and Run Triage steps so the dashboards reflect the completed work.

Reports and Lists: run Reports at the end of a work session when updated summaries are needed. Run Lists only when those reference views need refreshing.

Dashboard tags

Use these tags in the Harvest record’s Tags field. The dashboard recognizes tags separated by a pipe, comma, or semicolon.

  • queue:pending_harvest — queues harvesting work. It places the record in To do and keeps it in the Triage harvest section, including records with irregular periodicity.
  • queue:pending_updates — queues record-update work. It places the record in To do.
  • queue:pending_review — queues a review. It places the record in To do and uses its due date as the review date.
  • due:YYYY-MM-DD — sets the due date for any pending queue tag. harvest_due:YYYY-MM-DD is also accepted. Without either, queued work is due today.
  • review:Ny — schedules an irregular record for review N years after its Last Harvested date; for example, review:2y.
