PASDA Metadata Directory Harvester

This runs the PASDA metadata-directory harvester using config/pasda.yaml. It reads the flat PASDA metadata XML directory, reuses unchanged cached XML when possible, re-downloads changed XML, builds download and GeoJSON inventories, matches metadata records to assets, and writes Aardvark, distribution, registry, and review CSVs.

This can take awhile to run. A full run may spend time listing PASDA download directories, checking thousands of XML files, deriving themes, and writing several output files.

Watch the log below for terminal-style messages. It will show inventory-building progress and each metadata XML filename as it is fetched, reused from the registry, or read from cache.
